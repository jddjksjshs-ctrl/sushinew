# 🍣 Sushi Boom — робочий сайт зі спільним меню

Сайт доставки суші з **єдиним сервером**: коли адміністратор додає страву, змінює ціну або фото — це бачать **усі відвідувачі** (а не лише той пристрій, де робилися зміни). Замовлення, акаунти користувачів і сповіщення теж зберігаються на сервері.

## Що працює

- 🛒 Кошик і оформлення замовлення (картка демо-форма або готівка)
- 👤 Особистий кабінет: реєстрація / вхід / історія замовлень (працює з будь-якого пристрою)
- 🔐 Прихована адмін-панель: додавання страв (з фото), зміна цін, видалення, керування статусами замовлень
- 🔔 Сповіщення адміну про нові замовлення (з автооновленням)
- 🔄 Меню в усіх клієнтів автоматично оновлюється кожні ~20 секунд

## Як запустити локально

Потрібен лише **Node.js 18+** (жодних `npm install` — залежностей немає).

```bash
node server.js
```

Потім відкрий http://localhost:3000

Змінити порт або адмін-пароль:

```bash
PORT=8080 ADMIN_PASS=мій_секретний_пароль node server.js
```

## Як зайти в адмін-панель

Адмінка прихована від звичайних користувачів. Зайти можна одним зі способів:

- відкрити адресу із `#admin` у кінці (напр. `http://localhost:3000/#admin`)
- натиснути `Ctrl/Cmd + Shift + A`
- тричі клікнути на логотип у шапці

Після цього треба ввести **пароль адміністратора** (за замовчуванням `admin`, змінюється через `ADMIN_PASS`).

## Як викласти в інтернет (щоб бачили всі)

Будь-який хостинг, що вміє запускати Node.js. Скрізь команда старту — `node server.js` (або `npm start`), сервер слухає порт зі змінної `PORT`.

### Render.com (безкоштовно, найпростіше)
1. Залий цю папку в GitHub-репозиторій.
2. На render.com → **New → Web Service** → підключи репозиторій.
3. Build Command: *(порожньо)*; Start Command: `node server.js`.
4. У розділі Environment додай `ADMIN_PASS` зі своїм паролем.
5. **Важливо:** додай **Persistent Disk** і вкажи шлях монтування `/opt/render/project/src/data`, або задай змінну `DATA_DIR` на шлях диска — інакше після перезапуску дані (меню/замовлення) скинуться.

### Railway.app
1. New Project → Deploy from repo (або `railway up`).
2. Start Command: `node server.js`. Додай Volume і змінну `DATA_DIR` на його шлях.
3. Додай змінну `ADMIN_PASS`.

### Свій VPS
```bash
node server.js
# для постійної роботи краще через pm2:
npm i -g pm2 && pm2 start server.js --name sushi-boom
```

## Де зберігаються дані

Усе лежить у файлі `data/db.json` (меню, замовлення, користувачі, сповіщення). 
- Резервна копія = просто скопіювати цей файл.
- На «ефемерних» хостингах (Render/Railway free) файлова система може очищатися при перезапуску — тому під'єднай постійний диск (Persistent Disk / Volume) і вкажи `DATA_DIR`, як описано вище.

## Структура

```
.
├── server.js        # Node.js сервер + REST API + JSON-база (без зовнішніх залежностей)
├── package.json
├── public/
│   └── index.html   # фронтенд (увесь інтерфейс)
└── data/
    └── db.json      # база даних (створюється автоматично при першому запуску)
```

## API (коротко)

| Метод | Шлях | Доступ | Опис |
|-------|------|--------|------|
| GET | `/api/menu` | всі | меню |
| POST | `/api/orders` | всі | створити замовлення |
| POST | `/api/auth/register` / `/api/auth/login` | всі | акаунт |
| GET | `/api/me/orders` | користувач | мої замовлення |
| POST | `/api/admin/login` | — | вхід адміна (пароль) |
| GET | `/api/admin/data` | адмін | усі замовлення/меню/сповіщення |
| POST/PATCH/DELETE | `/api/menu` `/api/menu/:id` | адмін | керування меню |
| PATCH | `/api/orders/:id` | адмін | статус замовлення |
